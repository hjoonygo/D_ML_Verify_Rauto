## 1. 화면 구성 (위 → 아래)

### ① 헤더
`Rauto2 · 실시간 백테` + 버전(`r1.0 · 260626_02 · :포트`) + 상태줄(모드·현재가) + 우측 `● 리플레이 / ● 라이브 운영` 뱃지.

### ② 🏆 챔피언 바 (금색 박스)
- `🏆 CHAMPION` + 챔피언봇 이름
- `실거래 OFF` 뱃지(페이퍼면 표시) · 포지션 side 뱃지(`L/S + 포지션 수익률%`, 대기면 `대기`)
- `진입 <가> / 현재 <가>`
- `실제 계좌 잔고 — USDT`(라이브·실거래 시 실잔고)

### ③ 🛑 Emergency 버튼
- `🛑 Gho, HyoungJoon! Emergency` — 탭 시 모달(안전중지 / 강제중지 / 재개).
- 긴급중지되면 버튼이 빨강 점멸 + `🛑 긴급중지됨 — 탭하여 제어`.
- ★**이 세션에서 그 밑 리플레이 버튼(⏸일시정지·▶재생·⏮처음·⏭끝·⏩속도)은 제거함**(폰 실운영엔 불필요).

### ④ 차트 패널
- 탭: `15m · 1H · 4H(기본) · 1D` · `EMA` 토글 · `📋 매매내역` 토글.
- **크기**: 세로 = 가로의 **85%**(`aspect-ratio 1/0.85`).
- **표시 기간(4H 기준)**: 캔들 **60개 = 정확히 10일**(`CHN=60`). (1D=60일, 1H=2.5일…)
- **패닝**: 마우스 드래그 / 폰은 화면 꾹누른 뒤(≈220ms) 밀기 = 거리만큼 시간이동 + 손 뗄 때 관성. 더블클릭/두번탭 = 최신으로 리셋.
- **매매내역 탭**: 선택 슬롯 전체 거래 표(`진입~청산 / 방향@가·수량 / 레버 / 수익−비용=실수익`). 실거래=USDT·가상=%. 드래그 스크롤(상한 2000).

#### ★차트 색 범례 (이 세션 확정 — 시인성)
| 색 | 의미 |
|---|---|
| 🔵 **쨍한 청색 `#22a7ff`** | **롱 진입 수평선 · 이익 수익률(P&L)** |
| 🩷 **짙은 핑크 `#ff2d78`** | **숏 진입 수평선 · 손실 수익률(P&L)** |

→ 청/핑크(포지션·손익)와 초록/빨강(캔들 방향)을 색으로 분리해 헷갈리지 않게 함.

### ⑤ 슬롯 리스트 (좌측 하단, 이 세션 재설계)
- 헤더 = `슬롯 · 예상(녹)/실현 레짐수익 · 챔피언 포함`.
- **1번째 줄 = 챔피언**(🏆 + 금색 볼드 이름). 이하 = 나머지 로딩봇.
- **정렬**: 챔피언 다음은 **현재 레짐 실현수익 순위**(높은 순).
- **각 줄 표시**: `[🏆/☑/·] 봇이름  [L/S]  예상+13.8%(녹색)  +2.1% 4일`
  - **예상수익(녹색)** = 현재 레짐의 `reg_monthly` 이름표값(36mo in-sample 상한·참고).
  - **실현수익 `+X.X% N일`** = 현재 레짐 에피소드(7일추세 연속구간) 시작 이후 실현 복리 % + 그 레짐 지속일수. 이익=녹색/손실=빨강. **거래 없으면 회색 `0.0% N일`**(빈칸 방지).
- 줄 탭 = 그 슬롯 선택(차트 오버레이·우측 비교에 반영).
- ★**이머전시로 챔피언 해제되면 = 챔피언 없음** → 챔피언 강조 없이 전체가 순위대로.

### ⑥ ⬇ Bot 로딩 모달 (이 세션 전면 재설계)
탭 = 좌측하단 `⬇ Bot 로딩`. **타이틀 없음**(옛 '전체 11종…' 안내문·'챔피언 시스템 슬롯' 제목 모두 삭제). 위에서부터:
- **챔피언 시스템 슬롯 = 최대 8칸**:
  - **1번 = 현 챔피언**: 🏆 + **샛노란 볼드 글씨 + 샛노란 테두리**. 평소 **체크박스 없음(잠김)**.
  - **2~8번 = 로딩된 봇, 최신 로딩순**(가장 최근 로딩이 챔피언 바로 밑 맨 위). `✅` 탭 = 제거.
  - **빈 슬롯** = 회색 점선 `· 빈 슬롯 N ·`.
  - ★순서는 매번 `/state.json`을 새로 읽어 정함(방금 로딩=최상단 보장).
- **아래 = 등록봇 대기열**(미로딩 봇): **최근 등록순(역순)** · **첫줄 글씨·바탕 1톤 다운**(로딩 슬롯과 구별) · 2칸 보이고 스크롤. 탭 = 로딩.
- **규칙**:
  - 슬롯 **최대 8개** — 8칸 다 차면 대기열 탭 시 `최대 8개` 알림(기존 봇 먼저 제거). 서버도 `MAX_SLOTS=8` 강제.
  - **챔피언은 1번만 잠금** — Emergency로 챔피언 중지(해제) 후에만 챔피언 슬롯에도 체크박스(`🏆☑`) 나타나 제거 가능. 나머지·대기열은 항상 로딩/제거 가능.
- 모달은 **화면 위쪽 정렬**(중앙정렬 아님) + 박스 98vh + 카드 촘촘 → 세로로 안 잘림.

### ⑦ 챔피언선정 Sys (차트 밑 diagbox)
`Win the CHAMPION` + `Regime: <현 레짐>` / `기대수익률 1위 <봇> +N%/월` / `현재 Regime 수익 1위 <봇> +N%/N일`. **M20 자격 풀(mdd≥−22)** 만 후보.

### ⑧ 📧 이메일 버튼
`Champ. Performance to Crew` — 챔피언 성과를 Gmail+텔레그램으로 크루 공유(시크릿 = `rauto_secrets.txt`).

### ⑨ 우측(넓은 화면) — 비교
`챔피언 ↕ 선택 슬롯` 비교 카드(자산곡선·장세별 PF·전체/최근1주 통계).

## 2. 조작 요약 (폰)
- 봇 로딩/교체 = `⬇ Bot 로딩` → 대기열 탭(로딩) / 슬롯 `✅` 탭(제거). 챔피언 교체는 Emergency 중지 후.
- 차트 기간 이동 = 꾹누른 뒤 밀기 / 리셋 = 두번 탭.
- 봇 선택 = 슬롯 줄 탭 → 차트 오버레이·우측 비교 갱신.
- 긴급 = `🛑 Emergency` → 안전/강제 중지·재개.

## 3. 배포 (이 GUI를 폰 본방에 반영)
1. `dashboard.html` 수정 → **PC dev(ai-ml)는 새로고침만**으로 즉시 반영(서버가 매 요청 디스크 읽음).
2. **AWS 본방 반영** = `git add/commit` → `git push origin HEAD:rfrauto` → AWS `rauto2_autopull.py`가 `git reset --hard origin/rfrauto` + `Rauto2Server` 재시작.
   - ★**전제**: AWS에 ⒜git 인증(origin URL에 PAT) ⒝**`Rauto2Deploy` 스케줄 태스크(3분마다 autopull 실행)** 가 있어야 자동. 하나라도 없으면 push해도 폰엔 안 뜸(§27 '안 뜸' 진단).
3. 확인 = `/state.json`(경로 `/state.json`, `/state` 아님) — PC `localhost:8788`, AWS `ec2amaz…`.

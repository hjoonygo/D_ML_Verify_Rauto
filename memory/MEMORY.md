## ★세션 시작 즉시 (매번 — 자동로드되는 이 줄부터 머리에 박는다)
1. **연속성 브리핑** + **★수행비서**(Work Order 레지스터 `00_WorkHstr/WORK_ORDERS_REGISTER.md`의 🔴미실행·🟡진행중을 "이거 하셔야 합니다" 자동 리스트) + **세션명 1줄 질문**("세션명 `___`, 회차?"). 세션ID=`YYMMDD_회차_세션명`, 회차=이 세션 N번째(직전 잇기 아님), 첫=01. Auto/bypass여도 필수, 주제 피벗 시 재확인.
2. **100% 한국어 · 고딩설명 · 통제우선.** 영어/퉁치기/속도우선 = 캡틴이 '속인다'고 판단 = 금지. 무엇·왜·어떻게·앵커영향 먼저 → 오케이 → 코드.
3. **무손상.** 모든 수치주장은 앵커(+1851.6%) 1원단위 증명을 보인 뒤에만 '검증'. 못 보이면 '미검증'.
4. **★MDD = 4단 게이트(CLAUDE.md §26, 캡틴 2026-06-26).** 탐색·알파상승 = MDD 무제한/−30/−25 최대수익(−20 족쇄로 가능성 자르기 금지). **MDD−20 = 챔피언 인증(실거래 자격) 게이트** — 미달이면 Rauto 로딩○·실거래✕. 모든 백테 격리마진 **강제청산 횟수** 의무 산출. 헤드라인은 수익률 유지(지침3 F).
5. **★알파 수익률 체크 = ETF후(2024-01-01+) 데이터만(캡틴 2026-06-28).** 2023은 다른 레짐(+1851% 앵커 부풀림) → 앞으로 알파 평가는 post-2024 ~28개월로만. 단일출처 [지침2 그룹D 260628_01]. (전체 36개월은 검증엔진 앵커 재현용으로만 보존.)
6. **★★수익률 헤드라인 = OOS(held-out) 기준만(캡틴 격노 2026-06-28).** 모든 성과 보고의 주인공 숫자는 **held-out OOS**로만. **in-sample·천장(M0~M20 레버최적)·레버최대 수치는 헤드라인 금지** — 꼭 필요하면 `(in-sample 상한·실전 아님)` 명시 후 보조로만. 예: REVoi COMBO = **held-out OOS(lev3) +2,121%**가 유일한 '수익률'(천장 +35만%·+322만%·+98018% 등은 헤드라인 폐기). 천장 숫자를 실전인 양 말하면 '사기'로 간주(캡틴). 항상 OOS 없으면 "OOS 미산출"로 정직표기.
7. **★★보고 형식 = 종합통계 + post-2024(ETF후) 매월통계 동반(캡틴 2026-06-28).** 모든 백테·결과·**Back2TV** 보고는 ①종합통계(전체) 다음에 ②**2024-01-01+(BTC ETF후) 매월 통계표**(월수익률·거래수·롱/숏)를 반드시 같이 붙인다 — §0 '매월 양수' 목표를 월단위로 점검. 통합 월별표를 '전체'+'post-2024' 두 뷰로. 단일출처 CLAUDE.md §19·§20.
8. **★★봇 이름 = '시장판세 버전' 접미(캡틴 2026-06-28).** 시장구조 대전환(예: 2024 BTC ETF)을 봇 식별이름 끝에 버전으로 단다. **현재 = `@ETF`**(post-2024 전용 백테·#5와 1:1). 모든 봇 NAME·레지스트리·리포트·이름표 = `(봇명)@ETF`(현재 `REVoi@ETF`). .py 파일명·식별자엔 `@` 금지(표시이름만). 단일출처 CLAUDE.md §16.
9. **★★슬립근사 '적용한 현실값' 병기·헤드라인(캡틴 2026-06-28).** 모든 종합·매월 표에 슬립0(낙관)+**시장청산 슬립근사 적용 '현실' 수익률 병기**, 헤드라인=현실값. 기본 근사=시장청산 **10bp**(조절가능, §15.4 '0~20bp 견고' 내). flash-crash 꼬리는 레버상한 방어(평균슬립 미반영). 진입 지정가=무슬립. 구현=`engines/veri_edge.py` slip_bp. 단일출처 CLAUDE.md §19. ★혼동금지: 캡틴 기준값 = **OOS test(held-out) × 현실슬립** 1개(train·천장·슬립0은 보조).

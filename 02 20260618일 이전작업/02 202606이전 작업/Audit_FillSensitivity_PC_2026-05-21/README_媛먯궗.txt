미래참조/체결가정 감사 — PC 실행 (체결순서 민감도)

[왜] 1·2단계 흑자(PF~1.2)가 '봉 안에서 어느 극값이 먼저 닿았는지'라는 알 수 없는
     가정에 의존하는지 측정. 의존하면 흑자가 부풀려진 것일 수 있다.

[클로드 환경(6개월) 결과 — 참고]
   TF   fav(최악)  heur(메인)  adv
   15m   0.995     1.342     1.347
   30m   0.984     1.316     1.319
   60m   0.986     1.317     1.321
  → heur(메인 러너 가정)는 낙관 아님(adv와 거의 같음). 그러나 추세전략 최악순서(fav)에선 ~1.0(본전).

[36개월에서 볼 것 — 이게 진짜 답]
  36개월 heur PF는 1.09~1.24였다. fav는 보통 ~0.3 낮으므로 36개월 fav는 0.8~0.9(적자)일 수 있다.
  → 이 러너로 36개월 fav 값을 직접 확인하라. fav가 1 미만이면:
     "흑자는 체결가정에 기댄 것 = 보수적으론 본전~적자" 라는 뜻.

[넣는 법] zip 5개 파일 전부 D:\ML\Verify 에 풀고
[실행]   python Run_Audit_FillSensitivity.py
[결과]   Audit_FillSensitivity.csv (TF별 PF_fav / PF_heur / PF_adv / worst / 판정)

[중요] 이건 미래참조 '버그'를 찾는 게 아니다(버그는 없음을 이미 확인). 봉 단위 데이터의
       체결 불확실성이 결과를 얼마나 흔드는지 재는 것. 실거래에서 '지정가(메이커)'로
       지정된 가격에 체결하면 이 모호성이 줄어든다(단, 어느 레벨이 먼저 닿느냐는 여전히 변수).

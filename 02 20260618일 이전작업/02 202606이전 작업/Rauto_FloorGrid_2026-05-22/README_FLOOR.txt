[질문] 폭주 5건만 자르고 나머지(회복하는 구멍거래)는 안 죽이려면 손절 ROE 몇 %?

[방법 — 2부]
 (1) 기준측정: 플로어 사실상 무제한(999%)으로 1회 -> 각 구멍거래의 '최대역행 ROE(MAE)' 분포.
     폭주 5건은 ROE 100%+(가격 +24~83%)에, 회복거래는 낮은 구간에 갈린다. 그 사이 빈 구간이 답.
 (2) 그리드: 파국손절 15/20/25/30%(ROE)로 각각 36개월 하락장 실행.
     각 손절폭에서: 컷 건수 / 잔여폭주 건수 / PF / 순익 / 연도별 PF.
 * 엔진=정정본(구멍 fib_stop=None 일 때만 플로어). 비율 50:50, 혁신1 ON, trigger15, ext0.65.

[실행법]
 1) 이 zip을 하위 폴더에 푼다. 예) D:\ML\Verify\Rauto_FloorGrid_2026-05-22\
    (Merged_Data_with_Regime_Features.csv 가 상위 D:\ML\Verify 에, feat_struct_8 컬럼 필요)
 2) python run_floor_grid_v1.py   (pandas, numpy). 약 1~2분(기준측정은 폭주가 90일 타느라 좀 걸림).

[화면에서 볼 것 — 판정]
 (1) [폭주 N건의 구멍MAE_ROE] : 5건이 정말 100%+ 인지 확인
 (2) [구멍 MAE 분포] 히스토그램: 회복거래(낮은 구간) vs 폭주(100%+) 사이 '빈 구간'이 어디인지
     -> 그 빈 구간에 손절을 두면 폭주만 잡히고 회복거래는 안 죽는다.
 (3) 그리드 요약표: '잔여폭주 0' 이면서 '컷건수 최소'인 손절폭 = 최적
     (컷이 많으면 회복할 거래까지 죽이는 것)

[결과물 - 회신]
 콘솔 전체(MAE 분포 + 그리드표) + FLOOR_grid_summary.csv + FLOOR_ref_noFloor.csv 를 zip 1개로.

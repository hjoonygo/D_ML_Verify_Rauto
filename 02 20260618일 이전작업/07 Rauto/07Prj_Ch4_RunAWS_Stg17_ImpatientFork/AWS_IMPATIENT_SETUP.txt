[격리 설계 — 충돌 0]
  · 폴더    : 라이브 C:\run_Rauto   ↔  분기 C:\run_Rauto_Impatient (별도)
  · 태스크  : 라이브 Rauto_Daily(00:10)  ↔  분기 Rauto_Impatient(00:20, 10분 시차)
  · 산출/상태: run_daily.bat이 RAUTO_DIR·RAUTO_OPS_STATE를 분기폴더로 강제 →
              ops_state.json·paper_ledger·scorecard·health 전부 분기폴더에만 쓴다.
              (ops_common.rauto_dir()도 C:\run_Rauto 폴백 제거됨 — 라이브 절대 미참조)
  · 입력    : C:\BinanceData (공통, 읽기전용 시장데이터 — 공유 안전)
  · 텔레그램: TELEGRAM_BOT_TOKEN/CHAT_ID 머신변수 공유(같은 폰 수신).
              분기 알림은 태그 [PAPER-IMP] 로 자동 구분(라이브는 [PAPER]).

[0. 설치] 본 폴더 전체를 AWS C:\run_Rauto_Impatient\ 로 복사.
   (필수: test_/check_/daily_health.py/run_daily.bat/ops_common.py/alert_telegram.py/
    alert_check.py/bots\ . 선택제외: ab_local_replay.py·diag_equiv.py = 로컬검증용 스크래치)

[1. 중복 방지 확인]
   schtasks /Query /TN "Rauto_Impatient" 2>nul
   → 있으면: schtasks /Delete /TN "Rauto_Impatient" /F

[2. 등록 — UTC 00:20(라이브 00:10과 10분 시차), 1일 1회]
   schtasks /Create /TN "Rauto_Impatient" /TR "C:\run_Rauto_Impatient\run_daily.bat" ^
     /SC DAILY /ST 00:20 /RU SYSTEM /RL HIGHEST /F
   (HOURLY로 돌리려면 라이브와 동일하게: /SC HOURLY /MO 1 /ST 00:20 로 변경)

[3. 즉시 1회 강제 실행 — 무인 완주 검증]
   schtasks /Run /TN "Rauto_Impatient"
   확인:
     type C:\run_Rauto_Impatient\stg14_health.log     ← 마지막 줄 "OK ..."면 정상
     type C:\run_Rauto_Impatient\rauto_daily.log      ← 단계별 스탬프
     폰 텔레그램에 [PAPER-IMP] 🟢 시작 + 일일요약 1장 도착 확인

[v1에서 뭐가 바뀌었나 — 왜 다시 드리나]
 v1은 진입을 '한 줄 사슬'로 이어 돌려서, 비율을 바꾸면 청산이 밀리고
 1분 뒤 진입이 겹쳐 '건너뛰기'가 났습니다(125 -> 85건). 그래서:
  (1) 진입 125건을 '각각 독립 거래'로 시뮬 -> 누락 0, 모든 비율이 같은 125건을 봄.
  (2) 진입 시점으로 점프 + 표 복사 제거 -> 분 단위였던 게 약 10~20초로 단축.
  (3) 기준 config를 원본 TradeLog와 '거래별 1:1 대조'해 재현을 진단.

[실행법 - 2단계]
 1) 이 zip을 하위 폴더에 푼다.  예) D:\ML\Verify\Rauto_GridD2_2026-05-22\
    (Merged_Data.csv 는 상위 D:\ML\Verify 에 그대로 - 자동탐색)
 2) python run_grid_D_v2.py        (pandas, numpy 필요. scipy 있으면 DSR/PBO 더 정확)

[가장 먼저 볼 것 - 재현 자가검증]
 실행 끝에:
   [재현 자가검증] config 50:50 / 혁신1 ON
     진입 ___건 (목표 125)   PF=___ (목표≈2.864)   순익=___$ (목표≈+15,712$)
     [거래별 대조] 원본과 매칭된 진입 ___/125 | 순익 거의일치 ___ | ...
 -> 진입 125 & PF≈2.86 & '거의일치' 다수면 충실 재현 = 다른 config 신뢰 가능.
 -> 진입<125 거나 PF가 멀면, 그 줄 그대로 회신 주세요(데이터/시점 점검).

[결과물 - 회신]
 GridD_summary.csv + GridD_trades_*.csv(8개) 를 zip 1개로 업로드.

[해석 가이드]
 최고 비율은 PF만 보지 말 것. DSR 높고(>0.95 근처) PBO 낮은(<0.5) 비율이 진짜.
 혁신1 ON/OFF 비교로 '눌림목 재설정'이 수익을 올리는지 확인.

[미래참조(Lookahead) 점검]
 - 진입은 봉마감(종가)에서만. 고정진입은 과거 실거래라 미래정보 아님.
 - 청산은 1분봉 4틱(시->고/저->저/고->종) 인트라바 판정.
 - OB 스캔은 진입 직후 1회만(미래봉 미참조). 그 외 청산식은 검증모델 그대로.

[지표 출처]
 DSR = Deflated Sharpe Ratio (Bailey & Lopez de Prado 2014): 8회 시도 다중검정 보정.
 PBO = Combinatorially Symmetric CV proxy: 과최적화 확률(8config라 경량참고, 본격은 단계 C).

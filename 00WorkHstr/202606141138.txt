2) 사용명칭 정의
   - OPVnN 베이스라인: 추세봇 거래에 dev=(진입가-POC)/ATR(7h봉) 기반 수량조절을 사후적용한 확정 사이징.
     동결값 = OPV0.25 / n(NMULT)0.6 / N(N_BOOST)1.0 / EXP1.75 / ret +900.0% / MDD -19.74% / CPCVp25 2.7156.
     (OPV·n·N은 CLAUDE.md §9 확정알파와 일치. EXP/수익이 §9의 EXP1.559/+827%/MDD-16.1%와 다른 건
      §9가 업트렌드숏컷 포함 최종챔피언이고, 본 best는 그 전 '순수 OPVnN 사이징' 단계라 정상. 동결대상은 OPV/n/N.)
   - rdir(POC 회귀방향) = -sign(dev). same = (side==rdir, POC로 회귀하는 거래) / oppo = (side==-rdir, POC서 멀어지는 모멘텀 거래).
   - 버킷(PO3 정의 동일): |dev| <=0.5 / 0.5~1 / 1~2 / >=2 ATR.
   - POC 컨텍스트피처(검증대상): PO3-H1이 찾은 'POC거리 버킷 비대칭'(0.5~2ATR 강회귀 / >=2ATR 연속·역전)을
     베이스라인의 단일임계(OPV0.25) 위에 얹는 사이징/필터 보조. 단독사용 금지(지침).
   - 델타(1줄) = meanR(oppo,>=2ATR) - meanR(oppo,0.5~2ATR). PO3가 옳다면 >0(원거리 모멘텀=수익 → 베이스라인의 전구간 동일축소가 실수).
   - 신뢰도: 95=직접측정 / 55=통계추론 / 15=가설 / 0=모름.

3) lookahead 차단

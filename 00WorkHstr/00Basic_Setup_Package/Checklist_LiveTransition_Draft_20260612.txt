[A. 계정·키 보안]
 1. 실거래용 서브계정 분리(본계정 자산과 격리) + 이체 권한 없는 API키.
 2. API키 권한 = 선물 거래만(Enable Futures). 출금 권한 절대 OFF. 읽기키/거래키 분리 보관.
 3. Elastic IP 화이트리스트 — AWS 고정 IP만 허용(키 유출 시에도 타지 발주 불가).
 4. 키는 환경변수/자격증명 저장소로만(파일·코드·로그 기록 금지 — Dauto read-only 원칙 계승).
 5. 키 롤오버 절차 문서화(유출 의심 시 폐기→재발급→화이트리스트 재등록 ≤10분 목표).

[B. 주문 모듈 (설계 토론 항목 — 코딩은 승인 후)]
 6. 신호→주문 변환: Signal(size_pct·lev) → 수량 반올림(stepSize)·최소명목(notional) 가드.
 7. 체결 방식: 시장가 vs 지정가+추격 — 슬리피지 실측(1주 p50 vs 1.7bp)으로 결정.
 8. 멱등성: 같은 신호 중복 발주 방지(신호ID·클라이언트오더ID), 재시작 시 미체결 조회 복원.
 9. 레이트리밋·거부 대응: 429/418 백오프, 주문 실패 시 재시도 N회 후 ★긴급 알림+정지.
10. 포지션 동기화: 봇 내부 상태 vs 거래소 실포지션 주기 대사(불일치 = 즉시 정지).

[C. 리스크 가드 (전부 '엔진 밖 독립 레이어'로)]
11. 하드스톱: 계좌 자본 피크 대비 -20% 도달 시 전 포지션 청산 + 봇 정지(수동 재가동만).
12. 단일포지션 가드: 봇당 1포지션(이전 청산 100% 확인 후 신규 — §2.1 규칙의 기계화).
13. 노출 상한: k=0.77 배분 후 합산노출 캡(예: 5.6x) 초과 발주 차단.
14. 킬스위치: 수동 1클릭 전체청산+정지(RDP 불필요 — 원격 트리거 방식 토론 필요).
15. 데이터 이상 가드: Dauto 수집 끊김 N분 초과 시 신규 진입 금지(보유분은 가드만 유지).

[D. 자금·운영]
16. 소액 기준 토론: 제안 $500~$1,000(레버 22× 시 명목 ~$11k~22k — 최소수량 제약 충족 확인).
17. 모니터링: 일일 health ★긴급 발생 시 알림 경로(메일/텔레그램 — 방식 토론).
18. 페이퍼 병행: 실계좌 가동 후에도 페이퍼 슬롯 유지 — 실측 vs 모델 괴리 추적.
19. 전환 게이트(토론안): 공식 1주 ①동치 7/7 ②★긴급 0 ③슬리피지 p50 ≤ 가정 5bp
    ④주간 P&L 부호가 백테 동일구간과 일치 — 4조건 충족 시에만 소액 전환 상정.
20. 롤백 기준: 실계좌 2주 내 MDD -10% 또는 모델 괴리(체결가 기준) 누적 50bp 초과 시

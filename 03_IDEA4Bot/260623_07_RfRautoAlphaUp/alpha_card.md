## 3. 진짜 새 검증가치 (우리가 안 한 것만)
1. **OI Spike 이벤트 프레임**: 우리는 oi_change를 *연속 IC*로만 봄(약함). 제미나이는 'delta z>2 = 드문 이벤트' 순간의 forward return(event study). **조건부 이벤트는 연속IC가 죽어도 살 수 있음** = 진짜 안 본 각도.
2. **Order Block(OB)**: 미검증. 단 ★웹근거 = "주관적 해석 의존 시 신뢰성 백테 불가"·repaint 위험. → 반드시 *객관적 정의*(swing high/low 기반, 실시간 생성, 미래참조0)로만.
3. **교집합 조건부**: OI Spike ∩ OB ∩ CVD. 단 표본붕괴 위험(우리 OI+Funding서 경험) → 표본 충분한 2팩터부터.

## 4. 선행연구 (웹 조건검색 2026-06-23)
- OI delta/spike: 예측력 "일부" 있으나 ★"sharp OI spike는 헤지(투기 아님)일 수 있다"는 caveat → 방향 모호. 청산존서 impulsive spike(변동성 위치 설명, 방향 아님).
- Funding extreme: 역추세 = "exhaustion(소진) 신호"(바닥/천장)로 작동, '즉시 하락 예측'보다 소진. = 우리 fund_slope 양극단 IC와 정합. **극단치 한정**.

## 5. 시나리오 (다양·가능성)
- **S1 (H1 원안: OI Spike ∩ OB ∩ CVD ∩ Funding 교집합)**: 표본붕괴 + OB주관성 = 가능성 **낮음(20~30%)**. 단 이벤트 프레임은 새로움.
- **S2 (OI Spike → 변동성/노출 게이트, 방향 아님)**: 웹근거(spike=변동성 위치)와 정합. 가능성 **중(40%)**. 진입 아닌 오버레이.
- **S3 (우리 검증조합 마무리: CVD흡수 사이징 + OI손절 롱전용)**: 이미 CPCV 통과(신뢰70). ★제미나이 H1보다 앞섬. 잔여 = 틱슬립·OOS. ⚠️단 베이스 성급왕이 트레일인플레로 반증([[champion-return-exit-fill-inflated]]) → IC는 유효하나 수익률 재검증 필요.
- **S4 (Funding/OI 극단 이벤트 역추세)**: fund_slope 양극단·oi_zscore 음수일관과 정합. 극단치 한정 가능성 **중(40~50%)**.
- **S5 (OB 객관화 + OI중첩)**: repaint 차단 객관 OB. 가능성 **미지** — 측정해야 앎.

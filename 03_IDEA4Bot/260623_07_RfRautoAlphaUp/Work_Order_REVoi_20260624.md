## ★과제 0 [최우선·통합] — 장세·변동성 판단 휩소필터 스텝업 (캡틴 2026-06-26, 월12% 목표)
과제 1·3·4를 하나의 통합 알파로: **휩소(whipsaw) 차단 + 방향성 포착**으로 MDD−20 + 월12% 동시 달성 시도.
1. **장세판단 로직** — 추세/횡보/변동성 레짐 분류(atr60·ADX·chop·OI z) → 진입 게이트(불리레짐 진입 솎기).
2. **진입수량 조절** — 레짐·변동성·신호강도로 size 가변(불리↓ / 유리↑). RautoCEX 사이징과 분리(봇=알파, Rauto=사이징 경계 §25).
3. **★장세변동성판단 스텝업(휩소필터버전)** — 청산 피보 스텝업에 레짐 + **변동성(OI 데이터)** 적용:
   불리레짐·OI충격(쏠림·청산캐스케이드) 시 스톱 **타이트**(휩소 빠른탈출), 유리·추세확정 시 **느슨**(이익추격). = 핵심 레버.
4. **휩소·방향성 필터(+α)** — 저변동 횡보·극단 롱숏쏠림·OI충격 구간: 진입 솎기 또는 빠른 청산. 방향성 확정 구간만 풀노출.
5. **+α 배합** — OB·POC 구조레벨 부분익절(P레버 이미 압도적, 지침2)·HTF→LTF 스텝업(과제6) 선택 배합.
- **완료조건(MDD 4단 게이트·§26):** ⓐ탐색=M0 무제한/M30 −30%/M25 −25% 최대수익 + 강제청산 횟수 산출(천장·가능성 확인) → ⓑ**챔피언 인증=held-out CPCV 표준6 전 폴드 MDD ≤ −20(M20) + p25 > 0 + 월평균 ≥ +12%(또는 그에 준하는 분기수익)**. 환각0(verify·bot_trust_gates) 통과.
- **검증:** Back2TV(§20) 공용 — 휩소필터 ON vs OFF 분기별 수익률·MDD 비교. opt-in 레버(끄면 기존동일 §15.1).
- **재평가:** 통과 시 §9 확정 알파 후보 승격 → held-out 재검증 → 라이브.
- ★정직 경계: 단일 레짐피처로 MDD−20 불가(지침2 레짐진단). **고변동=REVoi 최고수익이라 '고변동 컷'은 거꾸로=금지** — 휩소는 '저변동 횡보+OI충격 동시'가 표적.

## 과제 1 [통합됨→과제0] — MDD−20 리스크 해결
- **레짐인지 리스크컷**(고변동 구간 노출↓ 등)으로 CPCV 폴드 MDD를 −20 이내로.
- 완료조건: CPCV 표준6 **전 폴드 MDD ≤ −20** + **p25 > 0** 동시 충족.
- 재평가: 해결 후 held-out 재검증 통과 시 → §9 확정 알파 후보로 승격.

## 과제 4 [연구] — 미결제약정(OI) 성격 규명 → 구조레벨 적용 (캡틴 지시 추가)
- **OI(미결제약정)의 성격을 밝힌다**: 가격 **돌파/저지**에 OI가 어떻게 작용하는지(누적·청산·쏠림).
- 규명된 OI 성격을 **OrderBlock(OB)·POC(Point of Control)** 등 구조 레벨에 **결합** → **돌파/저지 확률**을 높임.

## 과제 5 [연구] — OB 대응 전략(돌파 실패 시 대안) (캡틴 지시 추가)
- OB 앞에서 **돌파 안 되면** → 일단 **그 OB 앞에서 수익 실현**(저지 베팅).
- **돌파되면** → **재진입해 짧게 먹기**(breakout 확인 후 재진입 스캘핑).
- = OB를 '저지/돌파' 두 시나리오로 모두 수익화하는 양면 전략.

## 과제 6 [연구] — HTF → LTF 스텝업 혼합 (캡틴 지시 추가)
- **HTF(상위TF) 피보 스텝업 이후 LTF(하위TF) 스텝업을 섞는** 다중TF 전략.
- HTF로 큰 흐름·평단을 잡고, LTF 스텝업으로 정밀 청산/추격 → 손익비·승률 동시 개선 가설.

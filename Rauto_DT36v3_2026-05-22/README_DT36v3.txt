[v2 -> v3 정정 — 왜 또 드리나]
 v2는 하드플로어를 '모든 거래에 항상' 작동시키는 잘못된 구현이었다. 그래서 보호스탑(OB스탑)이
 멀쩡한 정상 거래까지 2.4~3%에서 다 잘려 촙(5천 거래·대규모 적자)이 났고, 그 결과는 신뢰 불가.
 정정: 하드플로어는 '보호스탑(fib_stop)이 아직 없을 때(=구멍)'에만 작동.
   - 구멍(첫 OB 도달 전 역방향) -> 비상손절 작동 -> 폭주 차단
   - 정상 거래(OB스탑 잡힘) -> 기존 2.864 OB/피보 메커니즘 그대로 (안 건드림)
 즉 2.4/3/5/8%는 '구멍 구간 전용 비상 손절폭'이다(정상거래용 아님).

[실행법]
 1) 이 zip을 하위 폴더에 푼다. 예) D:\ML\Verify\Rauto_DT36v3_2026-05-22\
    (Merged_Data_with_Regime_Features.csv 가 상위 D:\ML\Verify 에, feat_struct_8 컬럼 필요)
 2) python run_downtrend36_v2.py     (pandas, numpy). 약 1.5~2.5분.

[화면에서 볼 것 — 판정]
 (1) 폭주(90일강제) 0건  -> 구멍 차단 성공 (데이터끝 경계 1건은 자동 제외)
 (2) 청산사유에 '구멍 하드플로어'는 소수만(진짜 구멍), 'OB엣지/피보'가 다수 -> 정상거래 보존됨
 (3) 요약표 PF/순익/MDD + 전연도PF>1 -> 어느 손절폭이 하락장에서 버티는가
     * v2의 촙(거래 수천 개)이 사라지고 거래수가 확 줄면 정정 성공.

[결과물 - 회신]
 콘솔 캡처(요약표+연도별) + DT36v2_summary.csv + DT36v2_*.csv 를 zip 1개로.
 (참고: 진입은 동전던지기라 기계적진입 자체 한계는 있음. 이건 '구멍 빼고 정상 메커니즘'의 진짜 성적.)
